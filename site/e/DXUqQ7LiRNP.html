1k ft — Deep-dive / try it

Every token you send to an LLM leaves a trail. The model stores a key and a value vector per token, in full precision. That’s the KV cache. At 32k tokens on Llama 70B, it’s 17 gigabytes of GPU memory per request, and it only grows. Engineers have been quantizing the cache down for years. 8 bits works. 4 bits mostly works. 3 bits always broke accuracy — until Google’s TurboQuant, accepted at ICLR 2026. The trick: rewrite each vector in polar form. The angle distribution is known, so you only keep the radius and a few bits of angle. That’s PolarQuant. A Johnson-Lindenstrauss random rotation collapses the rest to sign bits. Result: 3 bits per number instead of 16. Roughly 5x less memory.
