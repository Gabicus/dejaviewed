1. Getting Started ESSENTIAL

Installation

  1. Install Claude Code CLI: npm install -g @anthropic-ai/claude-code
  2. Run in any project: cd your-project && claude
  3. First-time setup: Claude walks you through API key or Claude Pro subscription auth
Pro tip: Claude Code runs inside your terminal with full file system access. It can read, write, and execute code. Start in a git repo so all changes are reversible.

Day-One Essentials

  • Create a CLAUDE.md in your project root — this is Claude's persistent instruction file
  • Use /help to see all available slash commands
  • Use /compact when context gets long — compresses conversation history
  • Ask Claude to explain code before asking it to change code — builds shared understanding
  • Claude reads your entire project structure. Point it at files, don't paste code.

Access Methods

  • CLI (Terminal): claude — full power, file access, git integration
  • Desktop App: Mac/Windows native app with same CLI capabilities
  • Web App: claude.ai/code — browser-based, no install needed
  • IDE Extensions: VS Code and JetBrains plugins
  • Browser-only chat at claude.ai Replaced by claude.ai/code with full agent capabilities

2. CLAUDE.md & Project Configuration ESSENTIAL

CLAUDE.md is the single most important file for Claude power users. It's loaded automatically every session and persists your rules, preferences, and project context.

File Hierarchy

  • ~/.claude/CLAUDE.md — Global rules (apply to ALL projects)
  • ./CLAUDE.md — Project root rules
  • ./src/CLAUDE.md — Directory-scoped rules (override parent)
  • All levels merge — more specific wins on conflicts

What Goes in CLAUDE.md

  • Behavioral rules: Output style, response length, comment policy
  • Project conventions: Tech stack, file patterns, naming conventions
  • Safety guardrails: What Claude should never do (delete prod, push to main)
  • Self-improvement rules: "When I correct you, add a rule to CLAUDE.md that prevents the mistake"
Boris Cherny pattern (S-tier): Plan-first, self-improving CLAUDE.md. Claude proposes a plan, user approves, Claude executes. After corrections, Claude updates its own rules. The CLAUDE.md evolves with the project.

What Does NOT Go in CLAUDE.md

  • Session-specific state (use primer.md instead)
  • Ephemeral context that changes daily
  • Code documentation (that goes in the code)

3. Skills, Hooks & MCP ADVANCED

Skills are reusable instruction sets that Claude loads on demand. Hooks are shell scripts that run automatically on events. MCP servers extend Claude's capabilities to external services.

Skills

  • Location: ~/.claude/skills/<name>/SKILL.md or installed via plugins
  • Invoke: /skill-name in conversation
  • Examples: /brainstorming, /systematic-debugging, /frontend-design
  • Create your own: Write a SKILL.md with instructions, triggers, and steps

4. Memory, Context & Cognitive Scaffolding ADVANCED

The three layers that separate casual Claude use from a persistent AI workspace:

Context (what's in the window right now)

  • CLAUDE.md files (auto-loaded)
  • Files Claude has read this session
  • Conversation history
  • /compact compresses when approaching limits

Memory (what persists across sessions)

  • File-based: ~/.claude/projects/<project>/memory/ — structured markdown files with frontmatter
  • MEMORY.md: Index file always loaded — pointers to individual memory files
  • Types: user (preferences), feedback (corrections), project (state), reference (external pointers)
  • Auto-memory: Claude saves corrections and preferences automatically for future sessions

Harness (the scaffold around the model)

  • primer.md: Session state snapshot. Written at session end, read at session start. "Where were we?"
  • CLAUDE.md + skills + hooks: The full behavioral harness
  • Cognitive scaffolding: Phase-based reasoning architecture (triage → evidence → divergent paths → pre-mortem → verify)
  • The harness turns a chat window into a persistent, self-improving agent
Key insight: Context, Memory, and Harness are three distinct disciplines. Most people conflate them. Context is what Claude sees right now. Memory is what it recalls from before. Harness is the system that orchestrates both.

5. Prompt Engineering ADVANCED

Prompt engineering isn't dead — it evolved into prompt architecture. Single prompts matter less; the system around them matters more.

Thinking Keywords

  • think — standard reasoning depth
  • think hard — extended reasoning
  • think harder — deep multi-path analysis
  • ultrathink — maximum reasoning budget

What Still Matters

  • Be specific: "Fix the auth bug" → "The JWT token expiry check in auth.py:42 uses < instead of <="
  • Give context: What you've tried, what you expect, what broke
  • Name the format: "Respond in a table" / "Give me a bulleted list" / "Keep it under 3 sentences"

Old Prompt Patterns (Superseded)

  • "You are a helpful assistant" role prompts → CLAUDE.md behavioral rules are more durable and specific
  • Few-shot example prompting for code → Point Claude at existing code in the repo — real examples > synthetic ones
  • Chain-of-thought "let's think step by step" → Built into thinking keywords (think/think hard/ultrathink)
  • Manual retry loops for quality → Skills like verification-before-completion automate this

6. Agent Architecture ADVANCED

Claude Code is itself an agent. But the real power is orchestrating multiple agents — parallel worktrees, subagents, and multi-agent workflows.

Parallel Worktrees

  • Run 10-15 Claude sessions simultaneously, each in its own git worktree
  • Each agent works on a separate branch/feature without conflicts
  • The Boris Cherny pattern: human reviews + merges, Claude builds in parallel

Subagents

  • Explore agent: Fast read-only search — find files, grep symbols
  • Plan agent: Architecture design without file modification
  • Feature agents: code-architect, code-explorer, code-reviewer
  • Custom agents: Define via plugin agent-creator

Workflow Orchestration

  • Workflows: Deterministic multi-agent scripts with agent(), parallel(), pipeline()
  • Fan-out patterns: Review across dimensions, verify adversarially, synthesize
  • Budget-aware: Scale agent count to token budget (+500k directives)
